## Authentication

Most endpoints require authentication via Supabase Auth. Include your access token in the Authorization header:

```
Authorization: Bearer <access_token>
```

See the [Authentication guide](/authentication) for details on obtaining tokens.

## API Sections

### Health & Monitoring

Monitor service health and metrics:

- `GET /api/healthz` - Basic health check
- `GET /api/health/detailed` - Detailed component status
- `GET /api/metrics` - Prometheus-compatible metrics

### Streaming

Real-time execution updates:

- `POST /stream/cursor` - Server-Sent Events stream for orchestrator execution
- `WebSocket /ws` - WebSocket connection for live updates

### Agent Sequences

Multi-stage feature development pipelines:

- `GET /api/agent-sequences` - List sequences
- `POST /api/agent-sequences` - Create new sequence
- `PATCH /api/agent-sequences/{id}` - Update sequence
- `DELETE /api/agent-sequences/{id}` - Delete sequence

### Agent Workflows

Custom agent configurations:

- `GET /api/agent-workflows` - List workflows
- `POST /api/agent-workflows` - Create workflow
- `POST /api/agent-workflows/{id}/execute` - Execute workflow

### Organizations

Multi-tenant organization management:

- `GET /api/organizations` - List organizations
- `POST /api/organizations` - Create organization
- `GET /api/organizations/{id}` - Get organization details
- `DELETE /api/organizations/{id}` - Delete organization

### Tasks

AI-powered task management:

- `GET /api/tasks` - List tasks
- `POST /api/tasks` - Create task
- `POST /api/tasks/smart` - Create task with AI generation
- `PATCH /api/tasks/{id}` - Update task
- `DELETE /api/tasks/{id}` - Delete task

### Memory

Semantic memory and knowledge management:

- `POST /api/memory/save` - Save memory
- `POST /api/memory/search` - Search memories
- `POST /api/memory/infer` - AI-powered memory suggestions

### Agents

Agent execution and management:

- `GET /api/agents` - List available agents

### GitHub Integration

GitHub repository management:

- `GET /api/github/repositories` - List repositories from webhook activity

### Deployments

Deployment tracking:

- `GET /api/deployments` - List deployments
- `GET /api/deployments/{id}` - Get deployment details
- `PATCH /api/deployments/{id}` - Update deployment
- `DELETE /api/deployments/{id}` - Delete deployment

## Rate Limiting

Currently, there are no rate limits enforced. This may change in production.

## Error Responses

All endpoints follow a consistent error response format:

```json
{
  "detail": "Error message or array of validation errors"
}
```

### Common Status Codes

| Status | Description |
|--------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request - Invalid input |
| 401 | Unauthorized - Missing or invalid token |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found |
| 422 | Validation Error |
| 500 | Internal Server Error |

## Pagination

List endpoints support pagination (where applicable) with query parameters:

- `limit` - Number of results (default: 50, max: 250)
- `offset` - Number of results to skip
- `order_by` - Sort field (e.g., `created_at`, `updated_at`)

## SDKs and Client Libraries

### Python

```python
import httpx

class GlyxClient:
    def __init__(self, base_url: str, access_token: str | None = None):
        self.base_url = base_url
        self.headers = {}
        if access_token:
            self.headers["Authorization"] = f"Bearer {access_token}"

    async def create_smart_task(self, selected_text: str, page_title: str | None = None):
        async with httpx.AsyncClient() as client:
            response = await client.post(
                f"{self.base_url}/api/tasks/smart",
                headers=self.headers,
                json={
                    "selected_text": selected_text,
                    "page_title": page_title
                }
            )
            return response.json()

# Usage
client = GlyxClient("http://localhost:8000", access_token="your-token")
task = await client.create_smart_task("Implement auth", "Security Guide")
```

### JavaScript/TypeScript

```typescript
class GlyxClient {
  constructor(
    private baseUrl: string,
    private accessToken?: string
  ) {}

  async createSmartTask(selectedText: string, pageTitle?: string) {
    const response = await fetch(`${this.baseUrl}/api/tasks/smart`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        ...(this.accessToken && {
          'Authorization': `Bearer ${this.accessToken}`
        })
      },
      body: JSON.stringify({
        selected_text: selectedText,
        page_title: pageTitle
      })
    });

    return response.json();
  }
}

// Usage
const client = new GlyxClient('http://localhost:8000', 'your-token');
const task = await client.createSmartTask('Implement auth', 'Security Guide');
```

## WebSocket Connection

Connect to real-time updates:

```javascript
const ws = new WebSocket('ws://localhost:8000/ws');

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('Received:', data);
};

ws.send(JSON.stringify({ type: 'subscribe', channel: 'tasks' }));
```
